## Overview

The `Autosearch` component is a text input that opens a popover with search results as you type. Unlike [Autocomplete](/docs/components/features/autocomplete) which shows all options upfront, Autosearch is designed for **search-first workflows** where you type to trigger external search (like API calls).

## Dependencies

```package-install
@datum-cloud/datum-ui
```

## Usage

### Basic

```tsx
import { Autosearch } from '@datum-cloud/datum-ui/autosearch'
import { useState } from 'react'

const [searchResults, setSearchResults] = useState([])
const [loading, setLoading] = useState(false)

const handleSearch = async (query: string) => {
  if (!query) {
    setSearchResults([])
    return
  }
  
  setLoading(true)
  const results = await searchUsers(query)
  setSearchResults(results)
  setLoading(false)
}

<Autosearch
  options={searchResults}
  value={selectedUser}
  onValueChange={setSelectedUser}
  onSearch={handleSearch}
  loading={loading}
  placeholder="Type to search users..."
/>
```

### With Debounce Control

Control the debounce delay for search queries:

```tsx
<Autosearch
  options={results}
  onSearch={handleSearch}
  searchDebounceMs={500}  // Wait 500ms after typing stops
  placeholder="Type to search..."
/>
```

### Auto-select Single Result

When only one result matches, it auto-selects:

```tsx
<Autosearch
  options={results}
  onSearch={handleSearch}
  placeholder="Search..."
  // Auto-selects when results.length === 1
/>
```

### Inside Dialog/Modal

The `modal` prop controls popover behavior when used inside dialogs:

```tsx
<Dialog>
  <DialogContent>
    <Form.Field name="assignee" label="Assignee">
      {/* Works by default with modal={false} */}
      <Form.Autosearch
        options={searchResults}
        onSearch={handleSearch}
        loading={loading}
      />
    </Form.Field>
  </DialogContent>
</Dialog>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `options` | `AutocompleteOption[]` | `[]` | Search results to display |
| `value` | `string` | -- | Controlled selected value |
| `onValueChange` | `(value: string) => void` | -- | Callback when selection changes |
| `onSearch` | `(query: string) => void` | -- | Callback when search query changes (debounced) |
| `searchDebounceMs` | `number` | `300` | Debounce delay in milliseconds |
| `placeholder` | `string` | `'Search for an option'` | Input placeholder text |
| `emptyMessage` | `string` | `'No results found.'` | Message when no matches |
| `loading` | `boolean` | `false` | Show loading spinner |
| `modal` | `boolean` | `false` | Whether the popover is modal (required inside Dialog/Modal) |
| `disabled` | `boolean` | `false` | Disable the component |
| `className` | `string` | -- | Root wrapper classes |
| `inputClassName` | `string` | -- | Input field classes |
| `contentClassName` | `string` | -- | Popover content classes |
| `selectedClassName` | `string` | -- | Selected value card classes |

### AutocompleteOption

Uses the same option type as Autocomplete:

| Property | Type | Description |
|----------|------|-------------|
| `value` | `string` | Unique identifier |
| `label` | `string` | Display text |
| `description` | `string` | Optional secondary text shown below label |
| `disabled` | `boolean` | Disable this option |

## Behavior

### Search Flow

1. User types in input field
2. After debounce delay (default 300ms), `onSearch` fires with query
3. Parent component performs search (e.g., API call)
4. Parent updates `options` prop with results
5. Popover shows results below input
6. User clicks result to select

### Selected State

When a value is selected:
- Input is replaced with a card showing selected option
- Card displays label and description (if available)
- "Clear" button appears to deselect

### Empty States

- **No query:** Popover hidden
- **No results:** Shows `emptyMessage` with AlertCircle icon
- **Loading:** Shows spinner in input field

## Accessibility

- Full keyboard navigation (Arrow keys, Enter, Escape)
- Screen reader announcements
- Focus management
- ARIA attributes
- Clear button keyboard accessible

## Autosearch vs Autocomplete

Use **Autosearch** when:
- You need to search an external data source (API)
- Results depend on user query (not pre-loaded)
- You want a search input UX (like Google)
- You have potentially large result sets

Use **Autocomplete** when:
- All options are available upfront
- You need built-in filtering
- You need grouped options
- You need virtualization for pre-loaded lists
