import { Transfer } from '@/components/ui'

## Overview

The `Transfer` component provides a dual-panel interface for moving items between "available" and "selected" states. Click items in the source panel to add them to selected, or click the X button in the target panel to remove them.

## Usage

### Standalone

```tsx
import { Transfer } from '@datum-cloud/datum-ui/transfer'
import { useState } from 'react'

interface User {
  id: string
  name: string
  role: string
}

const users: User[] = [
  { id: '1', name: 'Alice Johnson', role: 'Admin' },
  { id: '2', name: 'Bob Smith', role: 'User' },
  { id: '3', name: 'Charlie Brown', role: 'Admin' },
]

const [selected, setSelected] = useState<string[]>([])

<Transfer
  items={users}
  value={selected}
  onChange={setSelected}
  itemKey="id"
  itemLabel="name"
  itemGroup="role"
/>
```

### With Forms

The Transfer component integrates with the Form system:

```tsx
import { Form } from '@datum-cloud/datum-ui/form'
import { z } from 'zod'

const schema = z.object({
  assignedUsers: z.array(z.string()).optional(),
})

<Form.Root schema={schema} onSubmit={handleSubmit}>
  <Form.Field name="assignedUsers" label="Assigned Users">
    <Form.Transfer
      items={users}
      itemKey="id"
      itemLabel="name"
      itemGroup="role"
    />
  </Form.Field>
  <Form.Submit>Submit</Form.Submit>
</Form.Root>
```

## Features

### Simple Click-Based Selection

Click items in the source panel to add them to selected:

```tsx
<Transfer
  items={users}
  value={selected}
  onChange={setSelected}
  itemKey="id"
  itemLabel="name"
/>
```

**Interaction:**
- **Source panel (left):** Click any item to add it to selected
- **Target panel (right):** Hover over item to reveal X button, click to remove
- **Select All button:** Adds all filtered items from source panel
- **Clear All button:** Removes all selected items

### Search

Filter items in the source panel with built-in search:

```tsx
<Transfer
  items={users}
  value={selected}
  onChange={setSelected}
  itemKey="id"
  itemLabel="name"
  searchable={true}
  searchPlaceholder="Search users..."
/>
```

Search uses basic string matching with 300ms debounce for performance.

### Grouping

Group items by a common attribute:

```tsx
<Transfer
  items={users}
  value={selected}
  onChange={setSelected}
  itemKey="id"
  itemLabel="name"
  itemGroup="role"  // Groups by 'role' field
/>
```

Items are grouped in both source and target panels.

### Custom Extractors

Use functions instead of field names for complex data structures:

```tsx
<Transfer
  items={users}
  value={selected}
  onChange={setSelected}
  itemKey={user => user.id}
  itemLabel={user => `${user.firstName} ${user.lastName}`}
  itemGroup={user => user.department?.name}
/>
```

## Examples

### With Preselected Items

Start with some items already selected:

```tsx
const [selected, setSelected] = useState<string[]>(['1', '3'])

<Transfer
  items={users}
  value={selected}
  onChange={setSelected}
  itemKey="id"
  itemLabel="name"
/>
```

### Without Grouping

Disable grouping for a flat list:

```tsx
<Transfer
  items={users}
  value={selected}
  onChange={setSelected}
  itemKey="id"
  itemLabel="name"
  // Don't provide itemGroup
/>
```

### Custom Panel Titles

Change the source and target panel headings:

```tsx
<Transfer
  items={users}
  value={selected}
  onChange={setSelected}
  itemKey="id"
  itemLabel="name"
  sourceTitle="Team Members"
  targetTitle="Project Assignees"
/>
```

### Disabled State

Make the entire transfer read-only:

```tsx
<Transfer
  items={users}
  value={selected}
  onChange={setSelected}
  itemKey="id"
  itemLabel="name"
  disabled={true}
/>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `T[]` | -- | All available items |
| `value` | `string[]` | -- | Currently selected item keys |
| `onChange` | `(value: string[]) => void` | -- | Callback when selection changes |
| `itemKey` | `keyof T \| ((item: T) => string)` | -- | Extract unique key from item |
| `itemLabel` | `keyof T \| ((item: T) => string)` | -- | Extract display label from item |
| `itemGroup` | `keyof T \| ((item: T) => string \| undefined)` | -- | Extract group name from item (optional) |
| `searchable` | `boolean` | `true` | Enable search in source panel |
| `searchPlaceholder` | `string` | `'Search...'` | Search input placeholder |
| `sourceTitle` | `string` | `'Available'` | Source panel heading |
| `targetTitle` | `string` | `'Selected'` | Target panel heading |
| `enableSelectAll` | `boolean` | `true` | Show Select All / Clear All buttons |
| `modal` | `boolean` | `false` | Whether the popover is modal (required inside Dialog/Modal) |
| `disabled` | `boolean` | `false` | Disable all interactions |
| `className` | `string` | -- | Additional CSS classes |

## Form Integration

When used as `Form.Transfer`, the component automatically:

- Connects to the form field context
- Displays validation errors below the field
- Handles disabled state from form
- Validates against Zod array schema

```tsx
const schema = z.object({
  assignedUsers: z.array(z.string()).min(1, 'At least one user required'),
})
```

## Implementation Details

The Transfer component uses a simple click-based interaction:

- **Source panel (left):** Shows unselected items with search and "Select All" button
- **Target panel (right):** Shows selected items with "Clear All" button
- **Click interaction:** Click item in source to add, hover X button in target to remove
- **Search:** Only available in source panel, filters with 300ms debounce
- **Grouping:** Displays grouped items in both panels when itemGroup is provided

## Search Behavior

- **Debounce:** 300ms delay after typing
- **Matching:** Case-insensitive substring match
- **Scope:** Searches item labels only
- **Performance:** Optimized for large lists (tested with 1000+ items)

## Accessibility

- Full keyboard navigation with Tab and Enter
- ARIA labels for panels and controls
- Screen reader announces item count and selection changes
- Clickable items with proper focus indicators
- Select All / Clear All buttons for bulk operations
