## Overview

The `Form` component is a compound component library that provides a declarative API for building forms. It uses a **pluggable adapter system** -- you choose between Conform.js or React Hook Form as the form backend, while the component API stays the same. All sub-components are accessed through the `Form` namespace (e.g., `Form.Root`, `Form.Field`, `Form.Input`).

## Adapter Setup

Wrap your application root with an adapter provider. This tells all `Form.*` components which form library to use.

### Conform Adapter

```package-install
@conform-to/react @conform-to/zod zod
```

```tsx
import { ConformAdapter } from '@datum-cloud/datum-ui/form/adapters/conform'

function App() {
  return (
    <ConformAdapter>
      <YourApp />
    </ConformAdapter>
  )
}
```

### React Hook Form Adapter

```package-install
react-hook-form @hookform/resolvers zod
```

```tsx
import { RHFAdapter } from '@datum-cloud/datum-ui/form/adapters/rhf'

function App() {
  return (
    <RHFAdapter>
      <YourApp />
    </RHFAdapter>
  )
}
```

### Dependencies

| Dependency | Conform | RHF | Notes |
|---|---|---|---|
| `zod` (>=4) | Required | Required | Schema validation |
| `@conform-to/react` (>=1) | Required | -- | |
| `@conform-to/zod` (>=1) | Required | -- | |
| `react-hook-form` (>=7.55) | -- | Required | |
| `@hookform/resolvers` (>=5) | -- | Required | |
| `@stepperize/react` (>=4) | Optional | Optional | For `FormStepper` (separate import) |

All form library dependencies are optional peer dependencies -- you only install what your chosen adapter needs.

## Usage

The form API is **identical** regardless of which adapter you use.

```tsx
import { Form } from '@datum-cloud/datum-ui/form'
import { z } from 'zod'

const schema = z.object({
  name: z.string(),
  email: z.string(),
})

<Form.Root schema={schema} onSubmit={(data) => console.log(data)} className="max-w-md space-y-4">
  <Form.Field name="name" label="Name">
    <Form.Input placeholder="John Doe" />
  </Form.Field>
  <Form.Field name="email" label="Email">
    <Form.Input type="email" placeholder="john@example.com" />
  </Form.Field>
  <Form.Submit>Submit</Form.Submit>
</Form.Root>
```

## Examples

### With Validation

Zod validation runs automatically. Required fields show an asterisk and error messages appear below the input.

```tsx
const schema = z.object({
  username: z.string().min(3, 'Username must be at least 3 characters'),
  email: z.string().email('Please enter a valid email address'),
})

<Form.Root schema={schema} onSubmit={handleSubmit} className="max-w-md space-y-4">
  <Form.Field name="username" label="Username" required>
    <Form.Input placeholder="Enter username (min 3 chars)" />
  </Form.Field>
  <Form.Field name="email" label="Email" required>
    <Form.Input type="email" placeholder="user@example.com" />
  </Form.Field>
  <Form.Submit>Register</Form.Submit>
</Form.Root>
```

### With Select, Checkbox, Switch, and RadioGroup

All common input types are available as `Form.*` sub-components that automatically bind to the form context.

```tsx
<Form.Root schema={schema} onSubmit={handleSubmit} className="max-w-md space-y-4">
  <Form.Field name="country" label="Country" required>
    <Form.Select placeholder="Select a country">
      <Form.SelectItem value="us">United States</Form.SelectItem>
      <Form.SelectItem value="uk">United Kingdom</Form.SelectItem>
    </Form.Select>
  </Form.Field>
  <Form.Field name="role" label="Role">
    <Form.RadioGroup>
      <Form.RadioItem value="admin" label="Admin" />
      <Form.RadioItem value="editor" label="Editor" />
    </Form.RadioGroup>
  </Form.Field>
  <Form.Field name="terms">
    <Form.Checkbox label="I agree to the terms" />
  </Form.Field>
  <Form.Field name="notifications" label="Notifications">
    <Form.Switch label="Enable email notifications" />
  </Form.Field>
  <Form.Submit>Save</Form.Submit>
</Form.Root>
```

### Conditional Fields (Form.When)

Show or hide fields based on the value of another field.

```tsx
<Form.Field name="contactMethod" label="Preferred Contact">
  <Form.Select placeholder="Select method">
    <Form.SelectItem value="email">Email</Form.SelectItem>
    <Form.SelectItem value="phone">Phone</Form.SelectItem>
  </Form.Select>
</Form.Field>

<Form.When field="contactMethod" is="email">
  <Form.Field name="email" label="Email Address" required>
    <Form.Input type="email" placeholder="you@example.com" />
  </Form.Field>
</Form.When>

<Form.When field="contactMethod" is="phone">
  <Form.Field name="phone" label="Phone Number" required>
    <Form.Input type="tel" placeholder="+1 (555) 123-4567" />
  </Form.Field>
</Form.When>
```

### Field Arrays

Dynamically add and remove groups of fields with `Form.FieldArray`.

```tsx
<Form.FieldArray name="members">
  {({ fields, append, remove }) => (
    <div className="space-y-3">
      {fields.map((field, index) => (
        <div key={field.key} className="flex items-start gap-2">
          <Form.Field name={`members.${index}.email`} className="flex-1">
            <Form.Input type="email" placeholder="member@example.com" />
          </Form.Field>
          <Button type="danger" theme="outline" size="icon" onClick={() => remove(index)}>
            <TrashIcon size={14} />
          </Button>
        </div>
      ))}
      <Button type="secondary" theme="outline" onClick={() => append({ email: '' })}>
        Add Member
      </Button>
    </div>
  )}
</Form.FieldArray>
```

### Multi-Step Form (Separate Import)

Break a long form into validated steps. The stepper is a **separate import** to avoid pulling in `@stepperize/react` for basic forms.

```package-install
@stepperize/react
```

```tsx
import { FormStepper, FormStep, StepperNavigation, StepperControls } from '@datum-cloud/datum-ui/form/stepper'

const steps = [
  { id: 'account', label: 'Account', description: 'Set up credentials', schema: accountSchema },
  { id: 'profile', label: 'Profile', description: 'Personal details', schema: profileSchema },
  { id: 'preferences', label: 'Preferences', description: 'Choose your plan', schema: preferencesSchema },
]

<FormStepper steps={steps} onComplete={(data) => console.log(data)}>
  <StepperNavigation />

  <FormStep id="account">
    <Form.Field name="email" label="Email" required>
      <Form.Input type="email" placeholder="you@example.com" />
    </Form.Field>
    <Form.Field name="password" label="Password" required>
      <Form.Input type="password" placeholder="Min 8 characters" />
    </Form.Field>
  </FormStep>

  <FormStep id="profile">
    <Form.Field name="fullName" label="Full Name" required>
      <Form.Input placeholder="Jane Smith" />
    </Form.Field>
  </FormStep>

  <FormStep id="preferences">
    <Form.Field name="plan" label="Plan" required>
      <Form.RadioGroup>
        <Form.RadioItem value="free" label="Free" />
        <Form.RadioItem value="pro" label="Pro - $10/mo" />
      </Form.RadioGroup>
    </Form.Field>
  </FormStep>

  <StepperControls />
</FormStepper>
```

### Dialog Form

Render a form inside a dialog using `Form.Dialog`.

```tsx
<Form.Dialog
  open={open}
  onOpenChange={setOpen}
  title="Create a Resource"
  description="Add a resource to manage your services."
  schema={schema}
  onSubmit={(data) => {
    console.log(data)
    setOpen(false)
  }}
  submitText="Confirm"
  submitTextLoading="Creating..."
>
  <Form.Field name="name" label="Name" required>
    <Form.Input placeholder="e.g. My Resource" />
  </Form.Field>
  <Form.Field name="description" label="Description">
    <Form.Textarea placeholder="Optional..." rows={3} />
  </Form.Field>
</Form.Dialog>
```

### Render Function Pattern

Access form state directly using the render function pattern on `Form.Root`.

```tsx
<Form.Root schema={schema} defaultValues={{ name: 'John' }} onSubmit={handleSubmit}>
  {({ reset, isSubmitting, isDirty, isValid, isSubmitted, submitCount, dirtyFields, touchedFields }) => (
    <>
      <Form.Field name="name" label="Name" required>
        <Form.Input />
      </Form.Field>
      <div className="flex gap-2">
        <Button onClick={() => reset()} disabled={isSubmitting}>Reset</Button>
        <Form.Submit disabled={!isDirty || !isValid}>Save</Form.Submit>
      </div>
      {isSubmitted && <p>Submitted {submitCount} time(s)</p>}
    </>
  )}
</Form.Root>
```

## Key Props

### Form.Root

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `schema` | `ZodType` | -- | Zod schema for validation |
| `onSubmit` | `(data) => void \| Promise<void>` | -- | Submit handler with validated data |
| `defaultValues` | `Partial<z.infer<T>>` | -- | Default field values |
| `mode` | `'onBlur' \| 'onChange' \| 'onSubmit'` | -- | Validation timing |
| `children` | `ReactNode \| (props: FormRootRenderProps) => ReactNode` | -- | Form content or render function |
| `className` | `string` | -- | Additional CSS classes |

### Form.Field

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | -- | Field name (supports dot paths like `address.city`) |
| `label` | `string \| ReactNode` | -- | Field label |
| `description` | `string \| ReactNode` | -- | Helper text below the field |
| `required` | `boolean` | `false` | Show required asterisk |
| `disabled` | `boolean` | `false` | Disable the field |
| `children` | `ReactNode \| (props: FormFieldRenderProps) => ReactNode` | -- | Input or render function |

### FormStepper (from `./form/stepper`)

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `steps` | `StepConfig[]` | -- | Array of `{ id, label, description?, schema }` |
| `onComplete` | `(data) => void \| Promise<void>` | -- | Called when all steps complete |
| `onStepChange` | `(stepId, direction) => void` | -- | Called on step navigation |
| `initialStep` | `string` | -- | Starting step ID |

## Hooks

| Hook | Import | Description |
|------|--------|-------------|
| `Form.useFormContext()` | `@datum-cloud/datum-ui/form` | Access form metadata, fields, and submit/reset functions |
| `Form.useFormState()` | `@datum-cloud/datum-ui/form` | Access isDirty, isValid, isSubmitted, submitCount, dirtyFields, touchedFields |
| `Form.useFieldContext()` | `@datum-cloud/datum-ui/form` | Access current field metadata inside a `Form.Field` |
| `Form.useField(name)` | `@datum-cloud/datum-ui/form` | Access and control a specific field by name |
| `Form.useWatch(name)` | `@datum-cloud/datum-ui/form` | Watch a single field's value reactively |
| `Form.useWatchAll()` | `@datum-cloud/datum-ui/form` | Watch all field values |
| `useStepper()` | `@datum-cloud/datum-ui/form/stepper` | Access stepper context (current step, navigation) |

## Adapter Types

The adapter system exports types for building custom adapters:

```tsx
import type { FormAdapter, NormalizedFormInstance, NormalizedFieldState } from '@datum-cloud/datum-ui/form'
```

| Type | Description |
|------|-------------|
| `FormAdapter` | Interface for implementing a custom adapter |
| `NormalizedFormInstance` | Adapter-agnostic form instance returned by `useCreateForm` |
| `NormalizedFieldState` | Adapter-agnostic field state returned by `useField` |
| `NormalizedFieldMeta` | Base field metadata (id, errors, required) |
| `NormalizedFieldArray` | Field array operations (items, append, remove, move) |
| `NormalizedFormState` | Form-level state (isDirty, isValid, dirtyFields, touchedFields, etc.) |
| `CreateFormProps` | Props passed to `useCreateForm` |
