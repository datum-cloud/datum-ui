## Overview

The Sidebar system provides a full-featured navigation panel with collapsible states, responsive mobile support (via Sheet), keyboard shortcuts, and hover-to-expand behavior. Use `AppSidebar` for a ready-made sidebar with navigation items, or compose your own layout using the lower-level primitives.

## Dependencies

This component requires the following packages:

```package-install
motion @radix-ui/react-slot @radix-ui/react-collapsible @radix-ui/react-separator @radix-ui/react-dialog @radix-ui/react-tooltip
```

```tsx

<SidebarProvider defaultOpen>

      { title: 'Home', href: '/', type: 'link', icon: Home },
      { title: 'Dashboard', href: '/dashboard', type: 'link', icon: LayoutDashboard },
      { title: 'Settings', href: '/settings', type: 'link', icon: Settings },
    ]}
    title="My App"
    currentPath="/dashboard"
    collapsible="icon"
  />
  <SidebarInset>
    <div>Main content area</div>
  </SidebarInset>
</SidebarProvider>
```

### Collapsed State

Set `defaultOpen={false}` on `SidebarProvider` to start the sidebar in collapsed state.

```tsx
<SidebarProvider defaultOpen={false}>
  <AppSidebar navItems={navItems} title="My App" currentPath="/" collapsible="icon" />
  <SidebarInset>
    <div>Sidebar starts collapsed</div>
  </SidebarInset>
</SidebarProvider>
```

## Props

### SidebarProvider

Wraps the entire sidebar layout and provides context to all child components.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultOpen` | `boolean` | `true` | Initial open state of the sidebar |
| `open` | `boolean` | -- | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | -- | Callback when open state changes |
| `expandOnHover` | `boolean` | `false` | Expand sidebar when hovered in collapsed state |
| `expandBehavior` | `'push' \| 'overlay'` | `'push'` | How the expanded sidebar affects layout |
| `showBackdrop` | `boolean` | `false` | Show backdrop overlay when expanded in overlay mode |
| `className` | `string` | -- | Additional CSS classes |
| `style` | `CSSProperties` | -- | Inline styles |

### AppSidebar

A pre-composed sidebar with navigation, header, and toggle built in.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `navItems` | `NavItem[]` | -- | Navigation items to render |
| `title` | `string \| ReactNode` | -- | Sidebar header title |
| `currentPath` | `string` | -- | Current URL pathname for active state detection |
| `linkComponent` | `ElementType` | `'a'` | Component to render navigation links (e.g., Next.js `Link`) |
| `closeOnNavigation` | `boolean` | -- | Close sidebar after clicking a navigation link |
| `defaultOpen` | `boolean` | -- | Override initial open state from provider |
| `collapsible` | `'offcanvas' \| 'icon' \| 'none'` | `'offcanvas'` | Collapse behavior |
| `side` | `'left' \| 'right'` | `'left'` | Which side the sidebar appears on |
| `variant` | `'sidebar' \| 'floating' \| 'inset'` | `'sidebar'` | Visual variant |

### NavItem

The data structure for navigation items.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | -- | Display text |
| `href` | `string \| null` | -- | Navigation URL (null for groups) |
| `type` | `'link' \| 'group' \| 'collapsible' \| 'externalLink'` | -- | Item behavior type |
| `icon` | `LucideIcon` | -- | Icon component from lucide-react |
| `children` | `NavItem[]` | -- | Nested navigation items |
| `disabled` | `boolean` | -- | Disable the item |
| `hidden` | `boolean` | -- | Hide the item |
| `showSeparatorAbove` | `boolean` | -- | Show a separator line above this item |
| `showSeparatorBelow` | `boolean` | -- | Show a separator line below this item |
| `excludePaths` | `string[]` | -- | Sub-paths that should not activate this item |
| `tabChildLinks` | `string[]` | -- | Additional paths that activate this item |
| `onPrefetch` | `() => void` | -- | Called on hover for route prefetching |

## Sub-Components

The sidebar system includes many composable primitives for building custom sidebar layouts:

| Component | Description |
|-----------|-------------|
| `Sidebar` | The main sidebar container |
| `SidebarProvider` | Context provider managing open/close state |
| `SidebarInset` | Main content area wrapper alongside the sidebar |
| `SidebarHeader` | Top section of the sidebar |
| `SidebarFooter` | Bottom section of the sidebar |
| `SidebarContent` | Scrollable content area |
| `SidebarGroup` | Groups related menu items |
| `SidebarGroupLabel` | Label for a sidebar group |
| `SidebarGroupAction` | Action button within a group header |
| `SidebarGroupContent` | Content wrapper within a group |
| `SidebarMenu` | Menu list container |
| `SidebarMenuItem` | Individual menu item |
| `SidebarMenuButton` | Clickable menu button with tooltip support |
| `SidebarMenuAction` | Secondary action on a menu item |
| `SidebarMenuBadge` | Badge indicator on a menu item |
| `SidebarMenuSub` | Sub-menu container for nested items |
| `SidebarMenuSubItem` | Individual sub-menu item |
| `SidebarMenuSubButton` | Clickable sub-menu button |
| `SidebarMenuSkeleton` | Loading skeleton for menu items |
| `SidebarSeparator` | Visual divider between sections |
| `SidebarInput` | Search input styled for the sidebar |
| `SidebarRail` | Draggable rail for resizing/toggling |
| `SidebarTrigger` | Toggle button to expand/collapse |
| `useSidebar` | Hook to access sidebar state and controls |

### Keyboard Shortcut

The sidebar can be toggled with `Cmd+B` (macOS) or `Ctrl+B` (Windows/Linux).
