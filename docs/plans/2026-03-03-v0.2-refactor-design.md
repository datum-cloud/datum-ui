# datum-ui v0.2 Refactor Design

## Overview

Refactor datum-ui from its current alpha structure into a production-grade, plug-and-play design system. The goal: fast, lightweight, scalable, developer-friendly — with zero-config consumer experience.

## Migration Strategy

**Fresh Start** — new branch (`feat/v0.2-refactor`), rebuild scaffolding from scratch, migrate existing components in. Best for a pre-production design system at v0.1.

## Consumer Experience

### Minimal Setup (3 lines)

```tsx
import { DatumProvider, Button } from '@datum-cloud/datum-ui'

function App() {
  return (
    <DatumProvider>
      <Button type="primary" theme="solid">Click me</Button>
    </DatumProvider>
  )
}
```

No CSS import. No theme class. The provider handles everything.

### With SSR FOUC Prevention (Next.js / React Router)

```tsx
import { DatumProvider, ThemeScript } from '@datum-cloud/datum-ui'

export default function RootLayout({ children }) {
  return (
    <html>
      <head>
        <ThemeScript />
      </head>
      <body>
        <DatumProvider>{children}</DatumProvider>
      </body>
    </html>
  )
}
```

### Theme Control

```tsx
import { useTheme, ClientOnly } from '@datum-cloud/datum-ui'

function ThemeToggle() {
  const { theme, setTheme, resolvedTheme, systemTheme } = useTheme()

  return (
    <ClientOnly>
      <button onClick={() => setTheme(resolvedTheme === 'dark' ? 'light' : 'dark')}>
        {resolvedTheme === 'dark' ? 'Light' : 'Dark'}
      </button>
    </ClientOnly>
  )
}
```

### Import Patterns

```tsx
// Barrel import (convenience)
import { Button, Dialog, Popover } from '@datum-cloud/datum-ui'

// Per-component sub-path (optimized tree-shaking)
import { Button } from '@datum-cloud/datum-ui/button'
import { Popover } from '@datum-cloud/datum-ui/popover'
```

## Package Export Strategy

### Unified Facade

`@datum-cloud/datum-ui` is the single published package. It exports both branded components and shadcn primitives through the same import pattern. When both layers have a component with the same name, datum-ui's version wins.

### Proxy Pattern for Shadcn Primitives

```tsx
// src/popover/index.ts (thin proxy)
export { Popover, PopoverTrigger, PopoverContent } from '@repo/shadcn/ui/popover'
```

### tsdown Multi-Entry Build

```ts
// tsdown.config.ts
import { defineConfig } from 'tsdown'

export default defineConfig({
  entry: {
    index: 'src/index.ts',
    'provider/index': 'src/provider/index.ts',
    'button/index': 'src/button/index.ts',
    'popover/index': 'src/popover/index.ts',
    // ... one entry per component
  },
  format: ['esm'],
  dts: true,
  external: ['react', 'react-dom'],
  clean: true,
})
```

### package.json Exports Map

```json
{
  "exports": {
    ".": { "types": "./dist/index.d.ts", "import": "./dist/index.mjs" },
    "./provider": { "types": "./dist/provider/index.d.ts", "import": "./dist/provider/index.mjs" },
    "./button": { "types": "./dist/button/index.d.ts", "import": "./dist/button/index.mjs" },
    "./popover": { "types": "./dist/popover/index.d.ts", "import": "./dist/popover/index.mjs" },
    "./styles": { "default": "./src/styles/root.css" },
    "./styles/*": { "default": "./src/styles/*" }
  }
}
```

ESM-only output. No CJS.

## Provider & Theme System

### Architecture

Absorbed from `cloud-portal/app/modules/datum-themes` — a battle-tested theme module.

```
src/provider/
├── datum-provider.tsx      # wraps ThemeProvider + CSS side-effect import
├── theme-provider.tsx      # core theme logic (React context, localStorage, system detection)
├── theme-script.tsx        # inline <script> for FOUC prevention (SSR)
├── client-only.tsx         # hydration-safe wrapper
├── script.ts               # theme resolution (serialized into blocking <script>)
├── types.ts                # Theme, UseThemeProps, ThemeProviderProps
└── index.ts                # exports: DatumProvider, useTheme, ThemeScript, ClientOnly
```

### DatumProvider

Thin wrapper that pre-configures ThemeProvider with datum-ui defaults:

```tsx
import '../styles/root.css'   // side-effect CSS injection
import { ThemeProvider } from './theme-provider'

export function DatumProvider({ children, ...props }: DatumProviderProps) {
  return (
    <ThemeProvider
      attribute="class"
      defaultTheme="system"
      storageKey="datum-theme"
      enableSystem
      enableColorScheme
      value={{ light: 'theme-alpha', dark: 'theme-alpha dark' }}
      {...props}
    >
      {children}
    </ThemeProvider>
  )
}
```

### Theme Modes

| Mode | Behavior |
|------|----------|
| `system` (default) | Follows OS `prefers-color-scheme`, live-updates on change |
| `light` | Always light |
| `dark` | Always dark |

### Exports

| Export | Purpose |
|--------|---------|
| `DatumProvider` | Root provider — CSS + theme + dark mode |
| `useTheme` | Hook — `theme`, `setTheme`, `resolvedTheme`, `systemTheme` |
| `ThemeScript` | SSR FOUC prevention (blocking inline script) |
| `ClientOnly` | Hydration-safe wrapper component |

## CSS & Token Architecture

Three-layer system (unchanged, proven design):

```
Layer 1: Figma Tokens          (figma-tokens.css)
  Raw OKLCH values: --midnight-fjord, --glacier-mist-*, --canyon-clay, etc.

Layer 2: shadcn Semantic Base   (shadcn.css)
  Neutral defaults: --background, --foreground, --primary, etc.

Layer 3: Alpha Theme            (alpha.css)
  .theme-alpha: maps Figma tokens -> semantic tokens
  .theme-alpha.dark: dark mode overrides
```

### File Layout

```
packages/datum-ui/src/styles/
├── root.css                # main entry (imports everything)
├── fonts.css               # @font-face declarations
├── custom.css              # app-specific overrides
├── tokens/
│   ├── figma-tokens.css    # raw Figma design tokens (OKLCH)
│   └── brand-tokens.css    # reference palette
└── themes/
    └── alpha.css           # .theme-alpha + .theme-alpha.dark
```
